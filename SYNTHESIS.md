# HorizonIQ â€” Editorial Synthesis (Phase 1)

Grounded, auto-published signal copy generated by a multi-role LLM chain, gated
so nothing unverified ever reaches production. This document is the source of
truth for how it works, how to run and tune it, and how to debug it.

> **One-line mental model:** the pipeline finds *quantitative* trends; synthesis
> turns a genuinely-moving trend into *words*, but only if a writer drafts it
> from real evidence AND an independent verifier + code gates approve it.
> Anything short of that keeps the hand-written ("curated") editorial.

---

## 1. Why this exists

Curated editorial (hand-written strings in `lib/domain/mock-intelligence.ts`)
cannot scale to surface *new* developments â€” the core promise of the product.
Synthesis is the "built-in intelligence that frames words from signals." Phase 1
does this for the **existing** interests. Phase 2 (not built) adds discovery of
**new** topics. See Â§12.

**Design tenet â€” fail-closed.** Auto-publish has no human in the loop, so the
default at every step is *silence*: if a key is missing, evidence is thin, the
model errors, output is unparseable, a gate trips, or the verifier rejects â€” the
signal simply keeps its curated text. The system prefers showing nothing new
over showing something wrong.

## 4. The multi-role chain (in `orchestrate.ts`)

Order matters; the first failure wins and returns a rejection/skip.

1. **Gate 0 â€” worth attempting?** (no LLM call)
   - `samples.length >= minSamples` (default **5**)
   - `confidence >= minConfidence` (default **68**)
   - `direction` is `rising` or `falling` â€” a **flat/"stable" week is skipped** (keeps curated).
2. **Writer (LLM)** â€” drafts `{title, headline, summary, brief, forecast}` strictly from the source titles, in the Economist/Bloomberg/FT voice; told to name a *specific* development, ban filler, invent nothing, use no figures. For non-AI-native interests it must not frame the subject as AI.
3. **Parse** â€” robust JSON extraction (tolerates stray prose / code fences). Unparseable â†’ reject.
4. **Vague gate (code)** â€” reject if the headline matches filler (`remains|continues|stable|steady|operates|trend|â€¦`) or is < 3 words.
5. **Purity gate (code)** â€” for non-AI-native interests, reject if the text contains `FORBIDDEN_AI_PATTERNS` (`\bAI\b`, `LLM`, `agentic`, â€¦).
6. **No-numbers gate (code)** â€” reject if the prose contains `\d+%` or `$\d` (figures belong in metrics, not prose â€” anti-fabrication).
7. **Verifier (LLM, independent pass)** â€” re-reads draft vs evidence; sets `ok=false` if any claim is unsupported, invented, hyped, **not centrally about the subject** (e.g. an energy story is not a commerce signal), or vague. Only `ok=true` publishes.
8. **Accept** â€” returns the narrative + up to 5 source citations + model/provider.

### Script-level gates (in `pipeline-synthesize.ts`)
- **Skip interests with no score.**
- **Skip multi-signal interests** â€” if >1 curated signal shares an interest (e.g. `cloud-computing` = CUDA + Cloud Infrastructure, `artificial-intelligence` = Agentic AI + Spatial Computing), interest-level evidence can't tell them apart, so we don't synthesize them (avoids stamping identical text on distinct signals).
- **Cross-signal de-duplication** â€” if a new headline's significant-word Jaccard overlap with an already-accepted one is > 0.6, it's dropped as a duplicate (keeps the first).
- **Rate limiting** â€” 4 s sleep after each LLM-making interest (stays under free-tier limits).

## 5. Configuration & knobs (`lib/pipeline/config/synthesis.ts`)

| Knob | Default | Effect |
|---|---|---|
| `writerModel` / `verifierModel` | `llama-3.3-70b-versatile` (Groq), `gemini-1.5-flash` (Gemini) | Which model drafts/checks. **Update if a provider retires a model** (see Â§10). |
| `minSamples` | 5 | Higher = need more source evidence to attempt. |
| `minConfidence` | 68 | Higher = only stronger trends. |
| `temperature` | 0.2 | Lower = more literal/grounded. |
| `aiNativeInterests` | ai, cloud, cyber, robotics, quantum | Interests where AI language is allowed. Mirrors the purity guard. |
| `FORBIDDEN_AI_PATTERNS` | AI/LLM/agentic/â€¦ | Blocks AI framing leaking into non-AI subjects. |

Other tunables: `VAGUE_HEADLINE` regex and the `< 3` word floor in
`orchestrate.ts`; the Jaccard `> 0.6` duplicate threshold and `4000` ms
rate-limit sleep in `pipeline-synthesize.ts`; `max_tokens: 800` in `llm.ts`.

**To make output stricter/rarer:** raise `minConfidence`/`minSamples`, add words
to `VAGUE_HEADLINE`. **To loosen:** lower them. Strict is the safer default.

Reading the log:

re-run it repeatedly without re-ingesting.

## 9. Publishing model â€” how a synthesized signal actually goes live

Production shows whatever is in the **committed** `narratives/latest.json`:
- **Empty file / no entry for an interest** â†’ curated text (default).
- **An accepted entry** â†’ that signal's headline/summary/brief/forecast are
  replaced (the stable **title is kept**), and the tile shows the
  "Auto-synthesized" badge (`presentation.provenance = "synthesized"`).

So to publish deliberately: let CI (with the secret) generate and commit it, or
commit a locally-generated file you've reviewed. To *not* publish yet: empty the
file (`{"version":1,"generatedAt":"â€¦","narratives":{}}`) or delete it before
committing.

## 10. Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| `synthesis disabled` in logs | No `GROQ_API_KEY`/`GEMINI_API_KEY` | Add the key to `.env.local` (local) or the GitHub secret (CI). |
| `LLM HTTP 400 json_validate_failed` | (historical) Groq's strict JSON mode rejected valid-ish llama output | Already fixed â€” we dropped `response_format: json_object` and parse robustly. If it recurs, check `max_tokens` in `llm.ts` (truncation). |
| `LLM HTTP 404` / "model not found" | Provider retired the model name | Update `writerModel`/`verifierModel` in `synthesis.ts` (see console.groq.com/docs/models). |
| `HTTP 429` | Rate limit hit | Increase the `sleep(4000)` in `pipeline-synthesize.ts`, or run fewer interests. |
| Everything `skipped`, 0 published | Quiet week â€” nothing rising/falling with enough evidence | Correct behavior. Lower `minConfidence`/`minSamples` only if you want more (riskier) output. |
| Vague headlines published | Gates too loose or thin evidence | Add terms to `VAGUE_HEADLINE`; raise the word floor; raise thresholds. |
| Off-topic / drifted signal (e.g. energy story under commerce) | A source query pulled a tangential item; verifier should catch it | The verifier's "centrally about the subject" check handles most; tighten the source queries in `lib/pipeline/config/*-queries.ts`. This is what Phase 2 fixes structurally. |
| Duplicate signals across interests | Same story matched two interests | Cross-signal Jaccard dedup handles it (`> 0.6`). Lower the threshold to be stricter. |
| Build fails `stale-snapshot` on synthesized signals | Guard comparing synthesized text to curated | Already handled â€” the guard skips `provenance === "synthesized"`. If it fires, confirm that skip is present in `verify-domain-purity.ts` (Check D). |
| `Module not found: generated-signals` on Vercel | Vercel ran `next build` instead of `npm run build` | Ensure `vercel.json` `buildCommand` is `npm run build` (regenerates the artifact). |

## 11. Guardrails recap (why you can trust auto-publish)

1. **Grounding** â€” writer only sees real source titles; instructed to invent nothing.
2. **Two-model check** â€” independent verifier re-reads every draft against evidence.
3. **Code gates** â€” purity (no AI leak), no fabricated figures, no vague filler.
4. **Centrality** â€” verifier rejects off-subject stories.
5. **Dedup** â€” no repeated headlines across signals.
6. **Fail-closed** â€” any failure â†’ curated text.
7. **Build-time re-scan** â€” `verify:domain` re-checks synthesized text for AI leaks before shipping.
8. **Transparency** â€” an "Auto-synthesized" badge marks machine-written signals.

## 12. Known limitations & Phase 2

- **Fixed interests only.** Phase 1 synthesizes the ~20 curated interests. It does **not** yet discover a brand-new topic (e.g. a novel quantum result outside the interest list). That is **Phase 2: topic discovery** â€” cluster rising terms from raw source samples into new candidate signals, then feed the same orchestrator. Richer, more on-subject evidence is also what pushes copy from "good" to "great".
- **Free-model ceiling.** Small free models produce thin copy from thin evidence. Better evidence (Phase 2) matters more than a bigger model.
- **Multi-signal interests are skipped** (see Â§4) until per-signal evidence exists (Phase 2).
- **Source-query drift.** Off-topic items come from broad queries in `lib/pipeline/config/*-queries.ts`; tightening those improves relevance.
